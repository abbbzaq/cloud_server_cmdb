# 提交规范（海王星风格）

本仓库统一使用以下提交格式：

```
<type>: <subject>
```

示例：

- `feat: 新增IAM登录与会话接口`
- `fix: 修复新建用户无法登录问题`
- `docs: 更新CMDB功能文档完成度与联调说明`
- `refactor: 重构assets接口为函数式风格`

---

## 1. type 取值

- `feat`：新功能
- `fix`：缺陷修复
- `docs`：文档更新
- `refactor`：重构（不改变外部行为）
- `style`：代码格式调整（不影响逻辑）
- `test`：测试相关
- `chore`：构建、脚本、依赖、配置等杂项

---

## 2. subject 书写要求

- 使用中文，简洁明确，建议 10~30 字。
- 使用动宾结构，描述“做了什么”。
- 不要写无意义文案（如“update”“修改一下”）。

推荐写法：

- `fix: 修复云账号更新接口405错误`
- `feat: 增加审计日志按时间区间过滤`

不推荐写法：

- `fix: bug`
- `feat: update`

---

## 3. 一次提交建议

- 一次 commit 聚焦一个主题。
- 功能改动与文档改动尽量拆分提交。
- 大改动建议分步提交（先重构、再功能、再文档）。

---

## 4. 常用提交模板

```bash
git commit -m "feat: <功能描述>"
git commit -m "fix: <问题描述>"
git commit -m "docs: <文档变更描述>"
```

---

## 5. 发布前检查建议

- 本地 `manage.py check` 通过。
- 关键接口联调通过（至少登录、资产列表、审计列表）。
- 文档与接口行为一致（返回字段、路径、方法）。
